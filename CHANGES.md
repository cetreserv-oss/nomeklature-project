### Правила защиты листов

Для стандартных рабочих вкладок (`отправка марки` и `фиксация сведений о ввозе`) настроена защита:

- можно редактировать строки с данными;
- можно добавлять новые строки;
- можно вставлять строки внутри листа;
- можно удалять строки, начиная со второй;
- можно менять ширину столбцов;
- нельзя переименовывать или изменять заголовки столбцов;
- нельзя добавлять новые столбцы;
- нельзя удалять столбцы.

Для вкладок `TK КОРРЕКТИРОВКА отправки марки` и `LA КОРРЕКТИРОВКА отправки марки` настроена отдельная защита:

- можно менять только столбец `Статус`;
- нельзя вручную менять любые другие ячейки;
- нельзя добавлять новые строки вручную;
- нельзя вставлять строки вручную;
- нельзя удалять строки;
- нельзя добавлять новые столбцы;
- нельзя удалять столбцы;
- нельзя менять заголовки.

### Форматирование

- стиль рабочих вкладок копируется из исходных листов `TK` и `ЛА`;
- стиль вкладок корректировки копируется по заголовкам и ширинам колонок из `Запрос по маркам 1.4.xlsm`, лист `Отправка марок (номенклатура)`;
- сохраняются ширины столбцов, высоты строк, базовое форматирование и закрепление верхней строки;
- старые листы `TK` и `ЛА` из выходного файла удаляются.

### Техническая реализация

- Лист `Отправка марок (номенклатура)` перестроен из заглушки в рабочий staging-лист.
- На листе теперь есть 8 рабочих колонок:
  - `Номер заказа`
  - `Код УТ`
  - `Номенклатура`
  - `Заявление на выдачу ФСМ`
  - `Кол-во`
  - `Оклейщик`
  - `Поставщик`
  - `МИЛ`
- Добавлены две кнопки справа от таблицы:
  - `Подготовить запрос`
  - `Отправить запрос`
- Кнопка `Отправить запрос` пока оставлена заглушкой без бизнес-логики.

### Логика кнопки `Подготовить запрос`

- очищает и нормализует ввод заказов в колонке `A`;
- удаляет пустые строки и дубли;
- приводит заказы к тому же формату, что используется на листе `Отправить запрос по ФСМ`;
- очищает старые подготовленные данные в `B:H`, не трогая колонку с вводом заказов;
- загружает введенные заказы в refresh-пайплайн `Отправить запрос по ФСМ`;
- запускает общий refresh без промежуточного success-popup;
- выполняет обязательную проверку сопоставления `(КМ)` и `(новое)`;
- при любой ошибке:
  - переводит пользователя на лист `Отправить запрос по ФСМ`;
  - показывает сообщение только об ошибке;
  - прекращает выполнение макроса;
- при success:
  - завершает подготовку данных для staging-листа;
  - показывает пользователю итоговый popup `Готово`.

### Правила заполнения staging-таблицы

- Источник данных: лист `Отправить запрос по ФСМ` после полного refresh.
- Отбираются только те строки, которые относятся к введенным заказам.
- Группировка выполняется по ключу:
  - `Заказ`
  - `Заявление (КМ)`
  - `Код (КМ)`
- Для каждой группы суммируется `Кол-во (КМ)`.
- В staging-лист переносятся:
  - `Номер заказа` <- `Заказ`
  - `Код УТ` <- `Код (КМ)`
  - `Номенклатура` <- `Позиция (КМ)`
  - `Заявление на выдачу ФСМ` <- `Заявление (КМ)`
  - `Кол-во` <- сумма `Кол-во (КМ)`
  - `Поставщик` <- `Поставщик (КМ)`
  - `МИЛ` <- значение `МИЛ` из `Настройка`
- `Оклейщик` вычисляется по обработанному `Алкоотчету`:
  - если `Место оклейки ФСМ/ЧЗ = завод`, берется `Поставщик`;
  - если `Место оклейки ФСМ/ЧЗ = ТС` и `Транзитный склад оклейки` содержит одно валидное значение, берется `Транзитный склад оклейки`;
  - пустые, составные или невалидные варианты не заполняются;
  - если внутри одной пары `Заказ + Код` найдено несколько разных валидных значений `Оклейщик`, макрос завершается с ошибкой.

### Fail-fast проверки

- если в рамках одной агрегированной группы различаются `Позиция (КМ)` или `Поставщик (КМ)`, выполнение останавливается;
- если валидация на листе `Отправить запрос по ФСМ` не проходит, выполнение останавливается;
- результат в staging-лист записывается только после полностью успешной подготовки без частичного вывода.

Обновлены или добавлены:

- `build_zapros_po_markam_1_4.py`
- `vba_marki_1_4\A00MainModule.bas`
- `vba_marki_1_4\Z02NomenclatureRequest.bas`
